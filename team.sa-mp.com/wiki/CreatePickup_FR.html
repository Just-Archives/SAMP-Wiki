Description:

Cette fonction fait exactement la même chose que AddStaticPickup, à la différence qu'elle retourne un ID de pickup utilisable pour le détruire ou utilisable dans OnPlayerPickUpPickup.


Paramètres:
(model, type, Float:X, Float:Y, Float:Z, Virtualworld)
modelLe modèle de pickup.
typeLe type de pickup.
Float:XLes coordonnées X où créer le pickup.
Float:YLes coordonnées Y où créer le pickup.
Float:ZLes coordonnées Z où créer le pickup.
virtualworldLe monde virtuel du pickup. Utilisez -1 pour l'afficher dans tous les mondes.(MÊME NARNIA ?! O:)


Retourne:

L'ID du pickup crée, -1 en cas d'échec (limite de pickup atteinte).


Note

  • Le seul type de pickup qui peut être ramassé en étant dans un véhicule est le 14
  • Les pickups sont affichés et peuvent êtres ramassés par tous les joueurs
  • Il est possible que si DestroyPicup() est utilisé lorsqu'un pickup est ramassé, le pickup pourra être ramassé par plus d'un joueur, du à un bogue.
  • Certains pickup n'ont pas besoin de script pour agir: C'est le cas par exemple du pickup M4, qui donnera automatiquement au joueur qui le ramasse une M4 et un certain nombre de munitions. Pour des actions entièrement scriptés, le type 1 doit être utilisé. 


new pickup; // Crée une variable où stocker le pickup
 
public OnGameModeInit()
{
    pickup = CreatePickup(1242, 2, 1503.3359, 1432.3585, 10.1191, -1);
    // On crée un pickup d'armure et on store l'ID dans la variable pickup
    return 1;
}
 
// Later..
DestroyPickup(pickup); //Exemple d'utilisation de l'ID du pickup
[edit]

Fonctions Relatives

Les fonctions suivantes peuvent être utiles car elles sont indirectement ou directement liées a cette fonction.

  • AddStaticPickup: Ajoute un pickup statique.
  • DestroyPickup: Détruit un pickup.

Callbacks Relatives

Les Callbacks suivantes peuvent être utiles car elles sont indirectement ou directement liées a cette fonction.

  • OnPlayerPickUpPickup: Appelée lorsqu'un joueur ramasse un pick-up.
