Client

[edit]

J'ai l'erreur "San Andreas cannot be found"

Le programme San Andreas Mutiplayer a besoin de GTA San Andreas pour fonctionner. Il a aussi besoin que la version du dit GTA soit la 1.0, les autres versions (2.0 comme sur Steam) ne marcheront pas. Cliquez ici pour télecharer un patch qui vous permettra de revenir en 1.0

[edit]

Je ne vois aucun serveur dans la liste de serveur du client SA:MP

Dans un premier temps, assurez vous d'avoir bien suivis la procédure détaille dans le guide "Débuter avec SA:MP". Si vous avez bien suivis les instructions mais que vous ne pouvez toujours voir aucun serveur dans la liste, assurez vous que votre pare-feux autorise bien la connexion de SA:MP. En arisons du nombre de pare-feux différents actuellement existants, nous ne pouvons offrir plus d'aide quant à leurs configurations. N'oubliez pas de vérifier que vous possédez bien la dernière version du client SA:MP.

[edit]

Le jeu solo charge à la place de SA:MP

Dans la plus-part des cas, cela est du à l'utilisation d'une version incompatibles de GTA:SA ou à l'installation du client SA:MP dans le mauvais dossier - Pour rappel, SA:MP dois être installé dans le même dossier que GTA:SA. De plus, SA:Mp nécéssite GTA:SA v1.0 pour fonctionner et ne fonctionnera aps avec les versions 1.01 ou 2.0. Si vous possédez GTA:SA dans une version incompatible (1.01 ou 2.0), vous pouvez 'downgrade' votre jeu pour pouvoir jouer à SA:MP. Pour cela, cliquez ici pour téélcharger le patch de 'downgrade'.

[edit]

Lorsque je me connecte à un serveur, je reçoit le message "Unacceptable NickName"

C'est parce que vous utilisez des caractères non autorisé dans votre pseudo (utilisez 0~9, a~z, [], (), $, @, . et = uniquement), ou que votre nom est plus grand que la limite de caractère pour un pseudo (qui est actuellement de 25 caractères). ça peut aussi être causé parce qu'un joueur connecté possède déjà un pseudo identique. Aussi un serveur SA:MP tournant sous windows et ayant un uptime plus grand que 50 jours peut causer ce bug.

[edit]

Screen sticks at "Connecting to ip:port..."

The server could be offline, or if you can't connect to any server, disable your firewall and see if it works. If it does you must set up your firewall properly - go to its website to find out how. It could also be that you have an old version of SA-MP, download the latest version from the SA-MP download page.

[edit]

J'ai modifié GTA: San Andreas et SA:MP ne se charge pas

Si le jeu ne se charge pas, vous n'avez qu'à supprimer vos modifications.

[edit]

Lorsque je lance GTA depuis SA:MP, ça ne démarre pas

Supprimez le fichier gta_sa.set dans vos documents et assurez vous de ne pas avoir de cheats/mods.

[edit]

The game crashes when a vehicle explodes

If you have 2 monitors then there are 3 ways to solve this:

  1. Disable your 2dr monitor when you play sa-mp. (Maybe not so smart if you like to have the monitor on.)
  2. Set your Visual FX quality to low. (Esc > Options > Display Setup > Advanced)
  3. Rename your GTA San Andreas folder (e.g. to "GTA San Andreas2") (This often works, however sometimes it may stop working again so you need to rename it to something else.)

Server

[edit]

Server instantly crashes when started

Most commonly it's an error in your server.cfg file or your gamemode is missing. Check the server_log.txt file and the reason should be located at the bottom. If not, check crashinfo.txt file.

[edit]

Server is not working - firewall is disabled

You will need to forward your ports to allow players to join your server. You can forward your ports using the PF Port Checker. Download it from: www.portforward.com If the ports are not forwarded that means you have to open them in your router. You can check the router list at http://portforward.com/english/routers/port_forwarding/routerindex.htm

It has all the information on how to forward ports.

[edit]

'Packet was modified'

The error commonly shown as: 
[hh:mm:ss] Packet was modified, sent by id: <id>, ip: <ip>:<port>
happens when a player times out or is currently having connection issues.
[edit]

Players constantly getting "Unacceptable NickName" error but it is valid

If you are sure you use an acceptable name and the server runs on Windows then try to change the compatibility option of the samp-server.exe to Windows 98 and it should be fixed after a server restart.

Windows servers with high up time may also cause this issue to occur. This has been noticed of around 50 days of server up time. To resolve it, a reboot is required.
