Sub-versions

I'm not sure whether functions should be grouped via major version (e.g. 0.3c) or by sub-version, for example 0.3c R1, 0.3c R2, 0.3c R3 etc.

Obviously these are different releases, but I think it might be better to group them by major version. Thoughts? MP2 07:50, 18 April 2014 (UTC)

You mean like for adding them to categories? We could always work with subcategories. For example, Category:Added in 0.3z R2-2 which in itself is then part of Category:Added in 0.3z. --Vince 08:01, 18 April 2014 (UTC)
But how will it work with the AddedIn template? Which categories should it be added to? It could be added to both, for example 0.3z and 0.3z R2-2. MP2 08:08, 18 April 2014 (UTC)
Not sure. It seems redundant to place them in both categories, but on the other hand it'd be nice to have a complete list of what functions were added throughout a major version. --Vince 08:20, 18 April 2014 (UTC)
Well it can easily be changed in the future. As for initial versions e.g. the first version of 0.3z, it could be put in the 'R1' category. MP2 08:22, 18 April 2014 (UTC)
