Wymagania

  • Dostęp do internetu (Pełny 100 slotowy serwer zużywa gdzieś 4Mbps wysyłania)
  • Otwarty port UDP (Domyślnie 7777) | Poradnik
  • Statyczny adres IP byłby mile widziany, ale nie konieczny
[edit]

Ustawianie serwera

[edit]

Zdobywanie potrzebnych plików

Pierwszym krokiem do konfiguracji serwera SA:MP jest pobranie plików serwera Windows stąd. Znajdziesz aktualną wersję plików serwera Windows pod nagłówkiem Windows Server - kliknij jeden z linków do pobrania, aby rozpocząć pobieranie, należy umieścić plik gdzieś, gdzie można go łatwo zapamiętać.

[edit]

Konfigurowanie serwera

Po pobraniu plików, rozpakuj (za pomocą programu, którym można rozpakować pliki .rar) pliki (nie trzeba mieć je w określonym folderze, ponieważ serwer nie musi sprawdzić, czy masz GTA: SA i SA-MP zainstalowany). Teraz przejdź do edycji pliku server.cfg.

Znajdź server.cfg w folderze głównym plików serwera i otwórz go w Notatniku lub w innym edytorze tekstu. Ten plik powinien pojawić się dla was w formie czytelnej. Ten plik jest do konfiguracji serwera, tutaj możesz określić, czy lanmode jest włączony (on) lub wyłączony (off), ilość slotów, port serwera, nazwa hosta, mapy, filterscripty, URL i hasło do rcona.

[edit]

Odpalanie serwera

Po skonfigurowaniu serwera można go uruchomić. Aby go uruchomić wystarczy po prostu kliknąć samp-server.exe, a serwer będzie uruchomiony.

Trzymanie serwera online

Z ograniczoną ilością opcji anty-crashowania w SA-MP Server, istnieje duża możliwość otrzymania błędów segmentacji lub innych błędów generowanych przez (Pawn) mape. Spowoduje to wyłączenie serwera, a będziesz musiał uruchomić go sobie ponownie.

Ten program uruchomi samp-server.exe ponownie, jeśli serwer SA-MP crashnie. Wszystko, co musisz zrobić to zapisać plik tekstowy z końcówką bat o następującej treści: 

:Start
samp-server.exe
goto start

Umieść plik w katalogu, w którym znajduje się samp-server.exe i go uruchom. Wpisz exit w konsoli serwera, aby sprawdzić czy to działa.
