Wechseln der Spielmodi

[edit]

Während des Serverbetriebs

Wenn Sie den aktuellen Spielmodus während des Serverbetriebes wechseln möchten können Sie dies auf zwei Wegen machen.

  • Sie können Ihrem Server beitreten und in der Chatzeile folgende Befehle eingeben:

/rcon login passwort (um sich als Administrator einzuloggen)

/rcon changemode mode (um den Spielmodus zu wechseln)

  • Sie können auf Windowsebene im Serverfenster den Befehl /rcon changemode mode eingeben.

Bei beiden Methode ist 'mode' mit dem Namen des Spielmodus zu ersetzen. Beispiel: /rcon changemode lvdm (Las Venturas Deathmatch)

[edit]

Einen Server rund um die Uhr betreiben

Wenn Sie auf Ihrem Server nur einen einzigen Spielmdus verwenden, dann beachten Sie folgendes:

  • Öffnen Sie die Datei server.cfg
  • Suchen Sie die Zeile in der steht gamemode0 lvdm 1 und ändern Sie diese in gamemode0 lvdm 0
  • Speichern Sie die Datei und starten Sie den Server erneut
[edit]

Wechselnde Spielmodi

  • Öffnen Sie die server.cfg und suchen Sie die Zeile mit den Spielmodi
  • Um bspw. drei Modi hintereinander laufen zu lassen, geben Sie untereinander an gamemode0, gamemode1, gamemode2 ein. Die Zahl nach dem Namen des Modus gibt die Rundenzahl an, die im jeweiligen Modus ausgetragen wird.

Beispiel: 

gamemode0 rivershell 3
gamemode1 cng 1
gamemode2 lyse 4
  • Hierbei wird der Modus Rivershell dreimal, Cops n Gangs einmal und Local Yokel viermal ausgetragen.

' Sie können bis zu 16 Modi eintragen. Der letzte Modus wäre demnach gamemode15.

[edit]

Einen eigenen oder heruntergeladenen Modus laufen lassen

  • Öffnen Sie das Serverinstallationsverzeichnis (Bsp. .../Rockstar Games/GTA San Andreas/server)
  • Kopieren Sie die bereits kompilierte *.amx-Datei in das Unterverzeichnis /gamemodes des Servers
  • Benutzen Sie das RCON-Kommando um den Modus zu wechseln (2.1)
  • Alternativ tragen Sie in der server.cfg den neuen Modus ein (2.3)
[edit]

Filterscripte

Grundsätzlich wie das Starten einen eigenen Modus mit folgenden Ausnahmen:

  • Kopieren Sie die *.amx-Datei in das Unterverzeichnis /filterscripts des Servers
  • Fügen Sie in der server.cfg in der Zeile filtersripts ihr Script mit scriptname hinzu
